# AI_CONTEXT вЂ” Bobby Brain Viz

Live 3D knowledge graph for Kenny's Obsidian vault. Hosted on Vercel. Auto-updates from a GitHub Action in the vault repo.

## What This Is

Kenny has a personal Obsidian vault (`/mnt/c/obsidian`, private GitHub repo) that doubles as memory for his AI assistant Bobby. This repo renders the vault's note graph as a hosted, public-facing 3D visualization вЂ” beautiful enough for portfolio embed, accurate enough to show Bobby growing over time.

## Data Format вЂ” graph.json

```json
{
  "nodes": [
    {
      "id": "Projects/Bobby Brain Viz",
      "label": "Bobby Brain Viz",
      "folder": "Projects",
      "created": "2026-05-24",
      "degree": 3
    }
  ],
  "links": [
    { "source": "Projects/Bobby Brain Viz", "target": "Maps/Bobby" }
  ],
  "generated": "2026-05-24T12:00:00Z"
```

`degree` is computed during generation and stored in JSON (not recalculated client-side).

## Stack

- **Vite + React** (no SSR, fully static)
- **react-force-graph-3d** вЂ” physics simulation + Three.js rendering, hooks: `nodeThreeObject`, `linkThreeObject`, `postProcessingComposer()`, `onRenderFramePost`
- **Three.js** вЂ” custom node meshes, particle trails, UnrealBloomPass for bloom
- **Vitest + React Testing Library** вЂ” unit tests for pure logic (colors, dates, useGraphData)
- **pytest** вЂ” tests for gen_graph.py Python parser (in vault repo)
- **Vercel** вЂ” free tier, auto-deploy on push to main

## Visual Direction вЂ” Neural/Organic (locked)

- Background: `#050820` (deep navy)
- Node color: discrete buckets by degree
  - 0-2: `#7B8CDE` (blue-violet)
  - 3-8: `#C4A0E8` (soft purple)
  - 9+: `#F4C87B` (gold)
- Recently added nodes (< 7 days): +50% emissive intensity, doubled pulse amplitude
- Node size: `log(degree + 1) * 3` px radius
- Bloom: Three.js `UnrealBloomPass` (strength 1.2, radius 0.4, threshold 0.2) вЂ” **desktop only**
- Particles: 5 per link, white, opacity 0.6, drift at 0.003/frame
- Breathing pulse: `scale = 1 + 0.04 * sin(time * 1.2 + node.phase)` вЂ” time is elapsed seconds
- `node.phase` = random `[0, 2ПЂ]` assigned at load to stagger each node's cycle

## Key Implementation Decisions (from eng review)

1. **No @react-three/postprocessing** вЂ” it requires R3F. Use Three.js `UnrealBloomPass` directly via `postProcessingComposer()` getter on the ForceGraph3D ref.

2. **Mobile bloom detection** вЂ” detect `!WebGL2RenderingContext` or `navigator.userAgent` mobile pattern; skip `UnrealBloomPass` on mobile. Breathing + particles still run.

3. **warmupTicks={100}** on ForceGraph3D вЂ” runs simulation headlessly before first render. Graph appears settled, no settling jank.

4. **Stale closure pattern** вЂ” animation callbacks read from `graphRef.current` (useRef), not React state. Store graph data in both `useState` (for re-renders) and `useRef` (for animation loop).

5. **Particle state in useRef Map** вЂ” `Map<linkId, ParticleState[]>` stored in useRef. `linkThreeObject` populates it on creation. `onRenderFramePost` iterates the map each frame. When `linkThreeObject` returns an object, react-force-graph-3d skips its own link line rendering.

6. **Code splitting** вЂ” Graph3D wrapped in `React.lazy + Suspense`. Loading skeleton shows immediately; Three.js bundle (~1MB) fetches in parallel.

7. **Error state** вЂ” `useGraphData` returns `{data, loading, error}`. Error card rendered on fetch failure or schema validation failure (not just a spinner).

8. **Schema validation** вЂ” Validate graph.json shape in `useGraphData`: check `nodes` is array, `links` is array, each node has `id` and `label`. No Zod вЂ” inline checks.
